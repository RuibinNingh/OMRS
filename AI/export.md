# 导出（HTML）

错题清单导出为**自包含 HTML**（图片 base64 内联，单文件可拷给任何带浏览器的设备）。两种版本：**A4 打印版**（默认）与**屏幕阅读版**。后端只产数据 + 内联模板，**版面与长图切片全部交给浏览器**。

## 为什么是 HTML（而非 docx）

旧 docx 导出长期受两个问题困扰：超长题图被截断、双栏栏底大片留白。根因是 **Word/OOXML 是封闭的版面引擎**——你无法在生成时查询「这一栏还剩多少高度」，于是只能盲切盲排；再加上 **LibreOffice 与 WPS 渲染存在保真度差**，本地预览正常、用户机器上却跑偏。

HTML 把这两个问题一起消掉：**浏览器既是排版引擎、又是用户最终查看/打印的引擎**——所见即所打印，没有跨渲染器保真度差。长图切片用「读像素找白缝 + overflow 裁切同一张内嵌图」，在浏览器里完全可靠（没有 WPS 那个 `srcRect` 白底涂白 bug，且图只存一份）。**附带收益：移除了对 Pillow 的依赖，项目恢复零第三方依赖。**

## 后端职责（`omrs/exporting.py`）

只做四件事，全部纯标准库：
1. **读题**：`_load_export_questions()` 从 mastery CSV + 题目 `.md` 取题（与旧实现一致，未改）。
2. **解析**：`_text_to_blocks()` 把每段正文按行切成块——非空文字行→`{t:'txt'}`，`![[名]]`/`![](路径)` 嵌入→`{t:'img'}`。细粒度成块既保留排版语义，也让浏览器端在双栏里填得更紧。
3. **取图**：`_img_payload()` 用 `_find_image()` 定位、`_read_image_info()`（纯 `struct` 解析 PNG/JPEG/GIF 尺寸，无 Pillow）读出宽高，base64 成 data-uri。
4. **组装**：`_build_export_data()` 产出 `{meta, questions, feedback, answers}`；`_build_html()` 读 `export_templates/{variant}.css` 与 `.js`，把数据 JSON（`</` 转义防提前闭合脚本）与模板内联成单个 HTML。

对外入口 `export_schedule_artifact(vault, uids, session_id, export_format, include_answers)`：
- `export_format`：`'a4'`（默认）/ `'screen'`；为兼容旧调用，`'docx'`/`'word'`/`'html'`/空 一律按 `a4`。
- 返回 `(bytes, session_id, "OMRS-{sid}-{variant}.html", "text/html; charset=utf-8")`。

模板是包内资源，放在 `omrs/export_templates/`，可直接当普通 JS/CSS 文件编辑：

| 文件 | 作用 |
|---|---|
| `a4.css` / `a4.js` | A4 打印版样式 + 排版引擎（引擎与应用层合并） |
| `screen.css` / `screen.js` | 屏幕阅读版样式 + 逻辑 |

## A4 排版引擎（`a4.js`，浏览器端）

几何完全沿用旧 docx：A4、上下 0.5in、左右 0.25in、双栏、栏距 0.5in → 栏宽 ≈348.85px(9.23cm)、栏内高 ≈1026.52px(27.16cm)。流程：

1. **测真实高度**：每个内容块按栏宽塞进离屏测量容器，读 `getBoundingClientRect().height`。长文字题干的折行由浏览器算，无需估算。块的间距用 `padding`（非 `margin`）以杜绝外边距折叠，保证「测得高度 == 落位高度」。
2. **贪心填栏**：维护「当前页/当前栏/当前 y」，逐块往固定尺寸的 A4 双栏页里填；填满一栏换下一栏，两栏满则换页。`keepNext`（题头/小标题）若落在栏底 `ORPHAN`(56px) 内则整体推到下一栏，避免孤悬。
3. **长图切片**（核心）：
   - 整张能进当前栏剩余 → 不切。
   - **PACK**（图能进一整栏但进不了当前栏剩余）：仅当能找到干净白缝时，切一片把当前栏填满、余下顺到下一栏；**找不到干净缝就整段顺到下一栏，绝不为填栏切穿内容**。
   - **FORCE**（图高过一整栏，必须切防截断）：优先用当前栏剩余里的白缝切，否则按整栏切；极端情况（密排长图无任何白缝）在墨最少处切并**标红虚线 + 顶栏告警计数**。
   - 切口靠 `analyze()` 读像素：逐行墨量 → 自适应底噪（5 分位 floor + 宽度相关容差）→ 连续 ≥G 行安静即「缝带」，切在缝带中心。
   - 切片用 `.slice`（固定高 + `overflow:hidden`）裹同一张 `<img>`、负 `margin-top` 平移——即 `srcRect` 的 HTML 安全版：浏览器零渲染怪癖、图只存一份。
4. **页码**：因分页由引擎掌控，每页底部居中渲染 `i / 总页数`（打印可见，对应反馈表的「页码」列）。

顶栏（不打印）有「打印/导出 PDF」「显示切口」开关与状态（页数/排版耗时/切穿告警数）。

> 实测：同一份 16 题数据，旧 docx 11 页、栏底留白高达 65%/49%/30%；A4 HTML 压到 7 页、各栏留白个位数，长图跨栏切在不可见的白缝处，无截断。

## 屏幕阅读版（`screen.js`）

单栏滚动，**不分页、不切图**（图按响应式宽度整张显示）。功能：
- **答案默认隐藏**，每卡「显示答案」按钮 + 顶栏「显示全部答案」全局开关——便于主动回忆自测。
- **图片灯箱**：点任意图全屏放大，点击/Esc 关闭。
- **主题筛选 chips + 搜索框**：按科目过滤、按 UID/分类/标签/题面文字搜索；科目<2 时自动隐藏筛选。
- 题头显示难度徽标、科目分类标签、错因/关联批注。
