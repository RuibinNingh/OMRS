# 错题录入助手

把一张/几张题目截图，变成一条结构正确的 OMRS 错题。**视觉与转写由你（Claude）完成，落库由 `scripts/add_card.py` 完成**——脚本只做确定性的拼装，不做判断。

## 适用前提

- 在 OMRS 仓库根目录下运行（该目录同时含 `omrs/` 包与 `错题/` 库；OMRS 默认 `--vault .`）。
- 若 `Skills/` 不在仓库根目录，调用脚本时用 `--repo` 指向含 `omrs/` 的目录。

## 两条硬约定（已与用户确认，不要违反）

1. **难度恒为 5，不要估算。** OMRS 的 `Difficulty` 不参与自适应（`feedback.py`/`scheduling.py` 从不更新它），真正随表现变化的是 `EF` / `Mastery` / 间隔。所以初始难度只是优先级公式里的一个常数权重，统一填 5 即可。
2. **错因一律留空。** 录入时不填 `## 错因`。用户在做题时自己就记住了错因;只有少数反复出现的题，由他事后在 Obsidian 手动补。不要替用户编写错因。

## 流程

1. **先看库里已有的科目/分类/知识点**，把新题对齐到已有的，避免建重复（如已有「三角函数」就别再建「三角函数题」，已有「二倍角公式」就别再写「二倍角」）：
   ```bash
   python Skills/mistake-card-creator/scripts/list_taxonomy.py --vault .
   ```
   一把吐出全库的科目、分类（按科目分组）、知识点（去重，带出现次数）。第 6 步定分类/知识点时就照这份清单对齐。
   - 题多时可只看一科：`--subject 数学`。
   - `ls 错题/` / `ls 错题/<科目>/` 仍可用，但只能看到科目和分类——**知识点不在目录结构里**（只存在 frontmatter 与 CSV 的 `Knowledge_Tags` 列），所以要对齐知识点必须用本脚本。
2. **读截图**（可能多张：题干、答案/解析有时是两张）。直接 `view` 图片即可，你能看到内容。
3. **判断题干 / 答案分界，并剔除噪声**：忽略作业帮等 App 的顶部导航栏、底部工具栏、广告位、「视频讲解」缩略图、水印、用户头像等一切非题目内容。只保留题干和解析本身。
4. **裁切成两张图**（题干图 + 答案图），存临时文件。数学、物理、几何题里的图形必须保留为图片（无法靠文字转写）。一张截图含题干+答案时用 PIL 切开；本来就两张时各自另存即可：
   ```python
   from PIL import Image
   im = Image.open("shot.png")            # 你看图后判断分界 y
   w, h = im.size
   im.crop((0, 0, w, split_y)).save("/tmp/q.png")   # 题干
   im.crop((0, split_y, w, h)).save("/tmp/a.png")   # 答案/解析
   ```
5. **转写文字**：题干和解析的文字转成 markdown；所有数学公式用 `$...$`（行内）或 `$$...$$`（独立行），OMRS 导出与仪表盘都按此渲染。保持中文。纯图形步骤可以只放图、文字从简。
6. **定科目 / 分类 / 知识点**（难度固定 5）。分类名尽量对齐第 1 步看到的已有分类。
7. **落库**——调用助手脚本：
   ```bash
   python Skills/mistake-card-creator/scripts/add_card.py --vault . \
     --subject 数学 --category 三角函数 \
     --question-file /tmp/q.txt --answer-file /tmp/a.txt \
     --question-image /tmp/q.png --answer-image /tmp/a.png \
     --knowledge 二倍角公式 --knowledge 辅助角公式
   ```
   - 文字可用 `--question-text "..."` 直接传，或 `--question-file` 传文件（公式多时用文件更稳）。
   - 知识点 `--knowledge` 可重复;没有就不传。
   - 纯文字题省略 `--*-image` 即可。
8. **复述结果**给用户：UID、文件路径、科目/分类、知识点。并提醒难度按默认 5、错因已留空（需要时自行补）。

## 多题与批量

一次发来多道题就循环执行：每道题独立走第 2–7 步、各调一次 `add_card.py`（UID 自动递增，不会撞号）。

## 脚本做了什么（供排查）

本 skill 有两个脚本，都**复用** OMRS 自己的逻辑（`omrs.common` / `omrs.creation`），不另起平行路径：

`add_card.py`（落库，确定性后半段）依次：① 调 `omrs.creation.create_question()` 建骨架（自动编号 UID、建立科目/分类锚点、首次重建索引）→ ② 把题干图/答案图复制进 `错题/附件/`，命名 `<UID>-question.*`、`<UID>-answer.*` → ③ 填 `# 题目`、`# 答案` 两段并嵌入 `![[...]]`，`## 错因` 留空 → ④ 把知识点写进 frontmatter（`相关知识点` + `知识点/` 标签）→ ⑤ 再跑一次 `build_index()` 同步难度/标签/知识点到 `mastery_data.csv`。

`list_taxonomy.py`（只读，第 1 步用）：复用 `omrs.common` 的 `mastery_path()` / `load_csv()` 读 `mastery_data.csv`，汇总三套去重清单——科目、分类（带科目限定，同名分类按科目区分）、知识点（全库去重，因一个知识点常被多科共享）。默认人类可读，`--json` 给程序消费，`--subject X` 限定单科。只读不写，跑多少次都安全。
